Problem Statement

In modern campus printing setups, users rely on fragmented messaging channels like WhatsApp, Gmail, or USB drives to transfer files to operators.

This workflow causes severe waiting bottlenecks, exposes personal documents to privacy leakage on operators' local drives, and demands high manual overhead.

No direct queue tracking | Privacy leaks | Driver incompatibilities
✨

Key System Features

  • QR-Based Pairing Instantly links user session to targeted printer via browser scan.
  • HD Inline PDF Preview High-definition page rendering in web view using PDF.js library.
  • Integrated Payment Flows Automated billing with instant bKash tokenized API checkout or counter Cash.
  • Strict Print Buffer Security Checks MIME types and renames files with secure hashes to prevent security exploits.

Integrated Billing Workflow

bKash Wallet API

Uses tokenized checkout APIs for automated 3-stage validation: Wallet number inputs, OTP codes, and secure PIN processing.

Counter Cash Code

Jobs enter queue as Unpaid. Releasing the print command requires physical cash submission and operator dashboard confirmation.

Operator & User Flow

1
Scan QR pairing label

User scans the sticker on the targeted physical printer node.

2
Upload document & preview

Document is securely sent to cloud buffer and previewed in HD.

3
Settle checkout options

Configure pages/copies and settle billing via bKash/counter Cash.

4
Automated spooling

Local print daemon fetches document and spools silent printing.

Privacy Compliance: Temporary files stored in the print buffer directory on the server are permanently deleted immediately after the printer daemon completes execution.
